Μετάβαση σε Τοπικό Web Server (Local HTTP Server) & Δυναμική Επικοινωνία
Στόχος αυτής της τροποποίησης είναι να απαλλαγεί η εφαρμογή από την ανάγκη εγγραφής αρχείων KML στον τοπικό δίσκο (π.χ. στο φάκελο C:\EGSA2GE\). Αντ' αυτού, η εφαρμογή θα "σερβίρει" τα δεδομένα KML απευθείας από τη μνήμη της μέσω ενός ενσωματωμένου, ελαφριού Web Server που θα εκτελείται στο παρασκήνιο.

Επιπλέον, θα υλοποιηθεί το Camera Tracking (Σταυρόνημα), ώστε η εφαρμογή να λαμβάνει δυναμικά τις συντεταγμένες από το σημείο που κοιτάζει ο χρήστης στο Google Earth και να τις μετατρέπει ακαριαία σε ΕΓΣΑ87.

IMPORTANT

Αυτή είναι μια ριζική αλλαγή αρχιτεκτονικής. Θα αντικατασταθεί το σύστημα εγγραφής αρχείων (FileService) με έναν τοπικό εξυπηρετητή (HTTP Server) και θα προστεθεί το αμφίδρομο σύστημα επικοινωνίας.

User Review Required
Σταυρόνημα και UI: Όταν ο χρήστης κουνάει τον χάρτη στο Google Earth, οι νέες συντεταγμένες ΕΓΣΑ87 θα λαμβάνονται από την εφαρμογή. Η προτεινόμενη σχεδίαση είναι να ενημερώνονται αυτόματα τα πεδία Χ και Υ του UI, σαν να τα πληκτρολόγησε ο χρήστης (καθώς και να δημιουργείται ένα νέο μικρό πλαίσιο "Στόχος Google Earth"). Συμφωνείτε με αυτή την προσέγγιση ή προτιμάτε απλώς μια ένδειξη χωρίς να πειράζονται τα κύρια πεδία εισαγωγής;
Θύρα (Port) Server: Προτείνεται η θύρα 8000 (http://localhost:8000). Εάν η θύρα αυτή χρησιμοποιείται από άλλη εφαρμογή στο σύστημά σας, θα μπορούσαμε να βάλουμε μια τυχαία ελεύθερη, αλλά αυτό κάνει το "Στατικό KML" (το μόνο αρχείο που θα σώζεται στο δίσκο) να πρέπει να ξαναδημιουργείται κάθε φορά. Θα δοκιμάσουμε πρώτα τη σταθερή θύρα 8000.
